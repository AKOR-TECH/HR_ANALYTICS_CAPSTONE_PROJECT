# HR Analytics Capstone Project

## Project Overview

This project focuses on cleaning, transforming, analyzing, and visualizing an HR employee dataset using Microsoft Excel. The goal was to convert raw HR data into meaningful business insights and an interactive dashboard that supports decision-making.

---

## Objectives

The objectives of this project were to:

- Clean and prepare raw HR data
- Identify and resolve data quality issues
- Create calculated fields for deeper analysis
- Perform exploratory analysis using Pivot Tables
- Build an interactive dashboard
- Generate actionable business insights

---

## Dataset Information

The dataset contains employee information including:

- Employee ID
- First Name
- Last Name
- Department Code
- Hire Date
- Salary
- Performance Score
- Employment Status

## Departments Included

- Finance
- Human Resources
- Information Technology
- Marketing
- Operations
- Sales

---

## Project Workflow

The project was completed in the following stages:

1. Data Exploration

## Data Cleaning process

### Initial Observations

During the initial exploration of the dataset, several issues were identified:

- Duplicate records
- Missing values
- Inconsistent text formatting
- Mixed date formats
- Inconsistent department codes
- Variations in employment status values
- Salary formatting inconsistencies

---

Data Issues Identified

Duplicate Records

Duplicate entries were found in the dataset.
Duplicate records can create misleading analytical results and inaccurate calculations.


Missing values were identified in several fields.
Missing data can affect calculations and reduce analysis accuracy.

Inconsistent Formatting

Text values contained:
- Extra spaces
- Mixed uppercase and lowercase values
- Inconsistent naming conventions
This may cause duplicate categories during analysis.

Mixed Date Formats
Different date formats existed in the dataset.
Date calculations and time-based analysis could become inaccurate.

---

Data Cleaning Steps Performed

Step 1: Removed Duplicate Records

Duplicate records were identified and removed.

---
Step 2: Handled Missing Values
Missing values were reviewed and treated appropriately
To improve reliability of analysis.

---
Step 3: Standardized Text Formatting

Actions taken:

- Removed unnecessary spaces
- Standardized capitalization
- Corrected inconsistencies
To improve consistency.

---
Step 4: Standardized Department Codes

- Converted codes into a uniform structure
- Corrected inconsistencies
To ensure proper grouping and reporting.

---

Step 5: Standardized Employment Status

Actions taken:
- Corrected inconsistent labels
- Applied consistent naming conventions
To improve reporting accuracy.

---

Step 6: Standardized Salary Formatting

Actions taken:
- Removed symbols
- Converted values into numeric format
To support calculations.

---

Step 7: Standardized Date Formatting

Actions taken:
- Converted all dates into a consistent format
To support time-based analysis.

# Key Business Insights

Workforce Distribution

- Information Technology had the largest workforce among all departments.
- Finance had the smallest employee population.

Attrition Analysis

- Employee exits were observed across departments.
- Certain departments experienced higher turnover rates than others.

Salary Analysis

- Average salary differed across departments.
- Human Resources recorded the highest average salary level.

Performance Analysis

- Employee performance varied across groups.
- Some performance categories contributed more to bonus allocation.

Years of Service Analysis

- Average employee tenure varied by department.

---

# Recommendations

- Investigate causes of employee attrition in departments with higher turnover
- Improve retention strategies
- Strengthen performance development programs
- Review compensation structures
- Continue workforce monitoring through dashboard reporting
